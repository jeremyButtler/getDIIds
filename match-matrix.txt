// This is an example of a match matrix you would supply
//   to alnSeq. In this case the match matrix is set to
//   the default settings.

// A match matrix is a matrix that is used to determine
// if two base pairs (or characters are a match. This is
// for the final output, not for the alignment step. Valid
// characters are a-z. However, if you compiled with
// -DWORDS, then every ascii character is valid

// This is a comment and needs to start at the start of a
//   line. It is ok to have stuff after the base
//   combination, since the read in step on looks at the
//   two base pairs and the provided number.  It does not
//   look past the number. However, starting a line with a
//   space is a good way to cause an error.

// A blank line will be ignored so long as it does not
//   have white space.
